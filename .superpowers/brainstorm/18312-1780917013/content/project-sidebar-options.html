A. 项目页替换全局侧边栏

进入项目后，左侧完全变成项目目录；点击“首页”回到全局工作台或项目列表。

优点

  • 最接近你给的截图
  • 项目内沉浸感强
  • 章节跳转最直观

代价

  • 需要调整 Layout 和项目页结构
  • 全局菜单在项目内不可直接见

B. 保留全局侧栏，再加项目目录侧栏

全局导航一直存在；项目打开后，内容区左侧多出一个项目目录。

优点

  • 全局导航不丢失
  • 改动较可控
  • 项目结构清晰

代价

  • 左侧占用更宽
  • 和截图相比层级更复杂

我目前的推荐

推荐 A：项目页替换全局侧边栏。它更符合你给出的截图，并且“项目详情 + 具体章节”的层级会更安静、清楚。实现时保留一个“首页”入口返回全局工作台。
